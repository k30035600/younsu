# Markdown → DOCX 빌드 (산출 기준: `docs/260404/`)

**정본 Markdown**은 `행정심판청구(최종)/` **직하**의 `260404_*_최종.md` 를 씁니다(하위 `260404/` 작업 폴더와 혼동 금지).  
**빌드된 DOCX·제출용 TXT·참고 추출 md**는 모두 **`docs/{yymmdd}/`**(현재 [`docs/260404/`](260404/)) 에만 둡니다 — **`행정심판청구(최종)/` 에 DOCX를 두지 않습니다.** **빌드 스크립트**는 **[`docs/scripts/`](scripts/)** 에 둡니다.

### 260404 이외 DOCX(별지·다른 날짜 접두)

- **`docs/scripts/` 파이프라인**은 `260404` 청구서·집행정지 등 **정해진 입력 MD → `docs/260404/`** 뿐입니다. **`260403_별지_…`처럼 다른 접두를 자동으로 `docs/`에 넣는 스크립트는 없습니다.**
- 별지·사실관계 정리표 등은 Word 등으로 만든 뒤 **`docs/<날짜>/`** 에 두면 산출물과 함께 관리하기 좋습니다.  
  예: `docs/260403/260403_별지_사실관계_시간축_정리표.docx`

## 제출·인쇄용 DOCX

**위원회 제출·프린터 인쇄**에는 [`docs/260404/`](260404/) 아래 `260404_*_최종.docx` 만 사용합니다. ([`docs/scripts/260404_build_final_docx.py`](scripts/260404_build_final_docx.py)가 python-docx로 생성.)

## 스크립트 (`docs/scripts/`)

- **[260404_build_final_docx.py](scripts/260404_build_final_docx.py)** — 제출용 맞춤(색·각주·끝./붙임 등). `**[증거자료 목록]**` 구간은 서두 안내 문단 뒤 **3열 표**(번호·호증·취지 / 파일·폴더)로 넣음.
- **[extract_reference_section.py](scripts/extract_reference_section.py)** — `## [참고]` **이하만** [`docs/260404/`](260404/)에 `*_참고추출.md`로 떼어 냄(별지·별첨용 원고).
- **[export_submission_txt.py](scripts/export_submission_txt.py)** — 제출용 `.txt`만 추출(아래 **제출용 원문만 TXT**).
- **[wrap_gab_cite_links_in_md.py](scripts/wrap_gab_cite_links_in_md.py)** — 정본 MD의 `갑 제N-M호증`를 `[갑 제N-M호증](#N-M)` 링크로 일괄 감쌈(포털 더블클릭·제출본 가독). `--apply` 없으면 건수만 출력.
- **입력 (MD 정본)**
  - `행정심판청구(최종)/260404_01_행정심판청구서_최종.md`
  - `행정심판청구(최종)/260404_02_집행정지신청서_최종.md`
- **출력 (DOCX)**
  - `docs/260404/260404_01_행정심판청구서_최종.docx`
  - `docs/260404/260404_02_집행정지신청서_최종.docx`

## 제출용 원문만 TXT

작성자 메모·`[참고]`·`> `·`*…*` 제외:

```powershell
python docs\scripts\export_submission_txt.py
```

→ [`docs/260404/`](260404/) `260404_01_행정심판청구서_제출용.txt`, `260404_02_집행정지신청서_제출용.txt`

## 실행

```powershell
# 둘 다
python docs\scripts\260404_build_final_docx.py

# 청구서만
python docs\scripts\260404_build_final_docx.py --01

# 집행정지만
python docs\scripts\260404_build_final_docx.py --02
```

**주의:** 생성하려는 `.docx`가 Word에서 열려 있으면 `PermissionError`로 저장이 실패합니다. 파일을 닫고 다시 실행합니다.

## 스타일 요약

| 항목 | 설정 |
|------|------|
| 글꼴 | 나눔명조(없으면 Word 기본) |
| 청구·신청 **원문(제출)** | **파란색**, 본문 **12pt**, `##` **14pt**, `###` **12pt**, 표제 **24pt** |
| **작성자 검토·검수·주의** | 줄 말머리 **`[작업] `·`[검토] `·`[검수] `·`[주의] `·`[편집] `** → **검정 10pt**, 접두 **그대로 표시**, 왼쪽 들여쓰기·앞여백으로 원문과 구분. 인용 `> `·`## [참고]…` 이하·말미 【판례 각주】도 **검정 10pt** |
| 본문 안 `*…*` | 파란 원문 안에서만 **검정 10pt(비이탤릭)** — 편집·부연. 제출 전 삭제·본문 흡수 권장 |
| 판례 각주 말미 | **검정 10pt** |
| 단락 정렬 | 제출 본문·소제목·작성자용 메모 단락 **양쪽 맞춤**. 증거 목록 **표** 머리는 **가운데**, 표 본문 셀은 **양쪽**. 말미 **【판례 각주】**는 **왼쪽**(번호 대응) |
| 여백 | 상·하·좌·우 mm 단위(공문서 길라잡이 취지에 맞춤, 스크립트 상단 주석 참고) |
| 판례 | 사건번호 패턴에 위첨자 번호 → 문서 말미 `【판례 각주】` 목록 |
| `**끝.**` | 단독 줄: 오른쪽 맞춤 |
| `**붙임** …` | 일반 단락(굵게 `붙임` + 내용). 청구서에는 **별첨 제1호**로 `(국가법령정보)판례모음` 폴더 전체를 명시함(전자제출·USB 동일 경로 권장). |
| `**청구인** …` / `**신청인** …` | **`**끝.**` 바로 다음**(빈 줄만 허용)에 오는 줄만 **서명란**으로 보아 오른쪽 맞춤. `## 당사자` 아래 당사자 표시는 **일반 본문**(양쪽 맞춤) |

## 제출본에서 빼거나 줄일 것(권장)

작성·편집용이지, **위원회 제출 정본에 꼭 넣을 필요가 없는** 요소입니다.

1. **`## [참고] 주요 인용 판례 및 적용 주석`**  
   - DOCX에 포함되어도 되나, 인쇄·전자제출 시 **해당 페이지를 삭제**하거나, 처음부터 MD에서 잠시 제외하고 빌드하는 방식이 흔합니다.

2. **PC 경로·파일명·스크립트명**  
   - 예: `행정심판청구(증거)/최종/(국가법령정보)판례모음/11_98두4061_폐기물처리업허가신청에대한불허가처분취소.pdf`, `260328_증명취지_….txt`, 과거 복사 스크립트명(`tools/archive/copy_junggong_photos_to_gab9.py` 등)  
   - **작성자 확인·USB 편철용**으로 MD에 둘 수 있으나, 제출용 본문에서는 **삭제하거나** “국가법령정보센터 사건번호 ○○” 정도만 남기는 편이 깔끔합니다.

3. **`*…*` 괄호 안 내부 메모**  
   - 일반교통방해·선거법을 “별 파일·별 폴더로 정리한다”는 식의 안내는 **취지만 한 문장**으로 줄이고, 경로는 제거하는 것을 권장합니다.

4. **`law.go.kr 원문 대조`**  
   - 위원회에 “원문과 맞추어 보라”는 뜻으로 둘 수도 있고, 제출 전 본인이 인용을 맞추는 **확인 메모**로만 쓰고 본문에서는 뺄 수도 있습니다.
